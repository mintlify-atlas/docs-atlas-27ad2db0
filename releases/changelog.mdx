This page tracks every release of the AWS Lambda PDF Generator, from the initial public release (v1.0.5) through the current stable version (v3.0.1). Each entry summarises what changed, why it changed, and — where relevant — what you need to do when upgrading.

## Version 3.0.1

This release continues the modernisation effort started in v3.0.0, advancing both the browser engine and the Lambda execution environment to their latest supported versions.

<AccordionGroup>
  <Accordion title="Major Changes">
    - Updated [Chromium](https://www.chromium.org/) to **version 122** (from version 116).
    - Updated [Puppeteer](https://pptr.dev/) to **version 22** (from version 21).
  </Accordion>
  <Accordion title="Technical Enhancements">
    - Updated AWS CodeBuild integration to use **Node.js version 20**.
    - Updated the Node.js Lambda runtime to **`nodejs20.x`** for all Lambda functions and layers.
    - Raised the minimum required version of the [Serverless framework](https://www.serverless.com/).
    - Updated dependencies implicitly.
  </Accordion>
</AccordionGroup>

<Note>
  If you are deploying from an existing v3.0.0 installation, run `serverless deploy` after updating your local dependencies. No endpoint or payload changes were introduced — the upgrade is a drop-in replacement.
</Note>

## Version 3.0.0

This release resolves a long-standing compatibility issue that prevented the service from running on Node.js 16 and Node.js 18, and brings the bundled browser engine up to Chromium 116.

<AccordionGroup>
  <Accordion title="Major Changes">
    - Updated [Chromium](https://www.chromium.org/) to **version 116** (from version 92).
    - Updated [Puppeteer](https://pptr.dev/) to **version 21** (from version 10).
  </Accordion>
  <Accordion title="Bug Fixes">
    - Resolved an inability to run on **Node.js 16 and Node.js 18** by replacing the unmaintained [`aws-chrome-lambda`](https://github.com/alixaxel/chrome-aws-lambda) package with [`@sparticuz/chromium`](https://github.com/Sparticuz/chromium). The `@sparticuz/chromium` package is actively maintained and provides a pre-compiled Chromium binary optimised for the AWS Lambda environment.
  </Accordion>
  <Accordion title="Technical Enhancements">
    - Updated the Node.js Lambda runtime to **`nodejs18.x`** for all Lambda functions and layers.
    - Raised the minimum required version of the [Serverless framework](https://www.serverless.com/).
    - Updated [Lerna](https://lerna.js.org/) by three major versions.
    - Updated other dependencies explicitly and implicitly.
  </Accordion>
</AccordionGroup>

<Note>
  The switch from `aws-chrome-lambda` to `@sparticuz/chromium` is an internal dependency change only. Your API requests, response shapes, and `serverless.yml` configuration remain compatible with v2.x — no changes to calling code are required.
</Note>

## Version 2.0.0

Version 2.0.0 is a major release that introduces breaking API changes, adds support for generating large PDFs, and restructures the project internals as a Lerna monorepo. **Existing deployments must be removed before upgrading.**

<Warning>
  **Breaking changes are present in this release.**

  - The `print/v81` and `print/v83` versioned endpoints have been **removed**. Only the unversioned `print` endpoint remains. Update any clients or integrations that call the old paths.
  - The application has been **renamed**. Deploying over an existing v1.x installation without first removing it will create a naming conflict. Run `sls remove` against your current deployment before deploying v2.0.0.
</Warning>

<AccordionGroup>
  <Accordion title="New Features">
    - **PDF files larger than 10 MB** are now supported. The service handles oversized responses in two ways:
      - **Gzip compression** — the response body is compressed before being returned, keeping the payload within API Gateway limits for moderately large documents.
      - **HTTP 303 redirect** — for documents that exceed the compressed threshold, the service returns an HTTP 303 (See Other) response pointing to a pre-signed S3 URL where the PDF can be downloaded directly.
  </Accordion>
  <Accordion title="Technical Enhancements">
    - Converted the project to a **[Lerna](https://github.com/lerna/lerna) monorepo** layout, separating the API package from shared tooling and making future multi-package development more maintainable.
    - Upgraded the [Serverless framework](https://www.serverless.com/) to **version 2.13.0** or later.
    - Added descriptive `description` fields for the application and individual functions in `serverless.yml`.
    - Updated the bundled Chromium version from **v83 to v88**.
  </Accordion>
</AccordionGroup>

<Note>
  **Migration steps for v1.x → v2.0.0:**

  1. Tear down the existing deployment: `sls remove --stage <your-stage>`
  2. Update your code to call `POST /print` instead of `POST /print/v81` or `POST /print/v83`.
  3. Deploy the new version: `sls deploy --stage <your-stage>`
  4. Update any clients that hard-code the old endpoint paths.
</Note>

## Version 1.1.1

This release improves observability and extends the request schema, with no breaking changes.

<AccordionGroup>
  <Accordion title="New Features">
    - Added a `source` field to the print request body. This optional string field lets callers tag requests with an identifier (e.g. an application name or team) for tracing and audit purposes.
  </Accordion>
  <Accordion title="Other">
    - Added typed printer failure categories to improve error classification and downstream error-handling logic.
    - Improved internal logging for easier debugging and monitoring of Lambda execution.
  </Accordion>

## Version 1.0.5

The initial public release of the AWS Lambda PDF Generator. This version establishes the core serverless architecture: an AWS API Gateway endpoint backed by an AWS Lambda function that drives a headless Chromium browser via Puppeteer to render HTML and produce PDF output.

<AccordionGroup>
  <Accordion title="Initial Public Release">
    - First publicly available version of the service.
    - Provides the `POST /print` endpoint for converting HTML to PDF.
    - Built on AWS API Gateway, AWS Lambda, [Puppeteer](https://pptr.dev/), and headless Chromium.
    - Deployed and managed via the [Serverless framework](https://www.serverless.com/).
  </Accordion>
</AccordionGroup>
